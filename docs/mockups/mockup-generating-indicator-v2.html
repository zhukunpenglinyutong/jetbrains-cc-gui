纯文字 + 闪烁光标，像 IDE 编辑器里正在输入的感觉。零装饰，最 JetBrains。

特点

  • 零边框零背景，完全融入消息流
  • 光标闪烁 = IDE 编辑器的"正在输入"隐喻
  • 改动量极小，只改文本样式
  • 缺点：视觉存在感弱，快速滚动可能被忽略

  • 不需要额外图标，文字本身即载体
  • 在消息流中存在感适中，不抢眼也不易忽略

C. 状态栏 Option

模仿 JetBrains 底部状态栏，用分隔区段展示状态、耗时、token 数。信息密度最高。

独立展示
正在生成响应
12s
1,247 tokens
完整信息变体
正在生成响应
1m 23s
3,891 tokens
streaming

特点

  • 与 JetBrains 底部状态栏视觉一致，天然适配
  • 信息密度高：状态 + 耗时 + token 一目了然
  • 可扩展：加入 model 名、cost 等区段
  • 缺点：在消息流中略显"重"，适合固定位置展示

左侧竖线 + 微妙背景 + 音频波形动画。类似 JetBrains 代码检查的侧边标记，辨识度高且不喧宾夺主。

推荐理由

  • 左侧竖线是 JetBrains 代码检查/warning/error 的经典视觉语言
  • 音频波形暗示"AI 正在说话"，隐喻自然
  • 无圆角胶囊，风格扁平，与 IDE 原生组件一致
  • 颜色可扩展：蓝色=生成 / 紫色=思考 / 橙色=排队
  • 背景极淡，不干扰消息阅读

E. 幽灵文字 Option

每个字依次呼吸明灭，像文字正在被"写入"。概念最贴近"生成"的本质。

独立展示
正在生成响应中…
12s

特点

  • 每个字依次明灭，视觉效果独特
  • 隐喻"文字正在被逐字生成"
  • 缺点：中文字符多时动画略慢，需要调整

F. 标签页 Option

模仿 JetBrains 编辑器标签页样式，状态指示器像一个正在活跃的 tab。

独立展示
正在生成响应 12s
↳ 响应内容将显示在此区域下方...

特点

  • Tab 样式是 JetBrains 最标志性的 UI 元素
  • 暗示"生成完成后这里会变成一个内容 tab"
  • 适合侧边栏或底部面板场景

G. 进度条 Option

顶部细进度条，类似浏览器页面加载。最轻量的"正在处理"提示。

独立展示
正在生成响应 12s
极简变体（仅进度条）

特点

  • 浏览器加载进度条的隐喻，用户天然理解
  • 极细（2px），几乎不占空间
  • 缺点：无文字信息，不知道"在生成什么"
